
## Баг-репорт по результатам тестирования

Дата тестирования: 2026-04-05  
Окружение: Windows 10, Python 3.11.8, pytest 8.3.3, Playwright 0.4.4 

---

### Баг №1. Фильтр по цене показывает объявления вне указанного диапазона

**Заголовок:**  
При установке диапазона цен (от 10000 до 50000) в результатах фильтрации присутствуют объявления с ценой выше 50000 ₽

**Серьёзность:** High  
**Приоритет:** High

**Описание:**  
Пользователь задаёт фильтр цены «от 10000 до 50000». Система должна показывать только объявления, цена которых находится в этом диапазоне (включительно). Фактически в выдаче присутствуют объявления с ценой 50179, 50445, 50895, 52480, 52566, 54736, 54764 ₽, что превышает верхнюю границу.

**Шаги воспроизведения:**  
1. Открыть страницу со списком объявлений.  
2. В поле "От" фильтра "Диапазон цен" ввести `10000`.  
3. В поле "До" фильтра "Диапазон цен" `50000`.  
4. Просмотреть цены в выданных объявлениях.

**Ожидаемый результат:**  
Все показанные объявления имеют цену от 10000 до 50000 включительно.

**Фактический результат:**  
Обнаружены объявления с ценой выше 50000 (например, 54764 ₽).

**Дополнительные материалы:**  
Лог теста:  
`AssertionError: Есть цены вне диапазона 10000-50000₽: [54764, 52480, 50895, 54736, 52566, 50179, 50445]`

### Баг №2. Фильтр по цене с отрицательной нижней границей возвращает объявления

**Заголовок:**  
При вводе некорректного диапазона цен (нижняя граница -1, верхняя 1000) система показывает объявления вместо пустого результата

**Серьёзность:** Medium  
**Приоритет:** Medium

**Описание:**  
Пользователь вводит в фильтр цены отрицательное значение в поле «от» (-1). Ожидается, что система либо не применит такой фильтр, либо покажет 0 объявлений (так как цена не может быть отрицательной). Фактически система выдаёт объявления с ценами от 0 до 1000.

**Шаги воспроизведения:**  
1. Открыть страницу со списком объявлений.  
2. В поле "От" фильтра "Диапазон цен" ввести `-1`.  
3. В поле "До" фильтра "Диапазон цен" ввести `1000`.  
4. Применить фильтр.

**Ожидаемый результат:**  
Результатов нет (0 объявлений) или фильтр не применяется с сообщением об ошибке.

**Фактический результат:**  
Показываются объявления с ценами (вероятно, от 0 до 1000).

**Дополнительные материалы:**  
Лог теста:  
`AssertionError: Ожидалось 0 объявлений для (-1, 1000), но найдены цены`

### Баг №3. Тогл «Только срочные» не фильтрует обычные объявления

**Заголовок:**  
При включении тогла "Только срочные" в выдаче остаются несрочные (обычные) объявления

**Серьёзность:** High  
**Приоритет:** High

**Описание:**  
На странице есть переключатель (toggle) "Только срочные". При его активации должны показываться только объявления с пометкой "Срочно". Фактически продолжают отображаться и обычные объявления.

**Шаги воспроизведения:**  
1. Открыть страницу со списком объявлений.  
2. Включить тогл "Только срочные".  
3. Просмотреть все показанные объявления.

**Ожидаемый результат:**  
Каждое показанное объявление имеет метку "Срочно".

**Фактический результат:**  
Среди результатов присутствуют объявления без метки срочности.

**Дополнительные материалы:**  
Лог теста:  
`AssertionError: После включения тогла показаны не только срочные объявления`

### Баг №4. Кнопка запуска автообновления не работает

**Заголовок:**  
При нажатии кнопки для запуска автообновления сообщение "Автообновление выключено" не исчезает, таймер не запускается и не отображается. Также не отображается прогресс-бар.

**Серьёзность:** Medium  
**Приоритет:** Medium

**Описание:**  
На странице присутствует функционал автообновления списка объявлений. При выключенном автообновлении отображается блок с текстом "Автообновление выключено". Нажатие на кнопку запуска автообновления должно скрыть этот блок и начать периодическое обновление. Фактически блок остаётся, автообновление не запускается.

**Шаги воспроизведения:**  
1. Открыть страницу со списком объявлений.  
2. Убедиться, что автообновление выключено (есть сообщение "Автообновление выключено").  
3. Нажать кнопку запуска автообновления.  
4. Проверить, исчезло ли сообщение, и началось ли обновление.

**Ожидаемый результат:**  
Сообщение "Автообновление выключено" пропадает, появляется индикация работающего таймера, отображается прогрес-бар, список обновляется через заданный интервал.

**Фактический результат:**  
Сообщение "Автообновление выключено" остаётся на месте, автообновление не происходит.

**Дополнительные материалы:**  
Лог теста:  
`AssertionError: Остался блок с сообщением 'Автообновление выключено'`
